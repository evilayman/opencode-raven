# Changelog

## 2.1.5

### Changed
- **On-demand MCP guidance** ‚Äî autogenerated guidance now preserves existing MCP order and appends newly loaded MCPs at the end.
- **Guidance descriptions** ‚Äî full-mode on-demand MCP guidance now keeps generated metadata intact while rendering only first-sentence descriptions to reduce main-model context.

## 2.1.4

### Changed
- **Subagent prompts** ‚Äî Raven no longer injects routing guidance into subagent prompts; global instructions and tool enforcement handle routing.

## 2.1.3

### Changed
- **Default timeout** ‚Äî Raven's default timeout is now 600s.
- **Provider errors** ‚Äî `raven_seek` now reports Raven provider/API retry failures and no-initial-response startup failures quickly instead of waiting for the full timeout.
- **Raven recovery** ‚Äî after a Raven provider failure, routed tools stop redirecting to `raven_seek` and tell the user how to switch Raven's model.

## 2.1.2

### Fixed
- **Local MCP TUI output** ‚Äî local on-demand MCP stderr is now piped and drained so server logs do not overlay the OpenCode TUI during startup checks or Raven MCP calls.

### Changed
- **OpenCode-style MCP config** ‚Äî on-demand MCPs now accept OpenCode-style `local` and `remote` entries, including local `command` arrays, `environment`, per-server `timeout`, and `enabled` flags.

## 2.1.1

### Changed
- **Context unit formatting** ‚Äî context estimates now switch from `K` to `M` after one million tokens for easier reading.
- **Loaded-only MCP guidance** ‚Äî autogenerated on-demand MCP guidance now includes only successfully loaded MCPs, keeping failed or pending servers out of main-agent instructions.
- **Startup guidance refresh** ‚Äî Raven now preserves existing autogenerated on-demand MCP guidance during startup and rewrites it only after MCP health checks update.

## 2.1.0

### Added
- **On-Demand MCPs** ‚Äî Raven can now connect to MCP servers from `onDemandMcpServers` internally through Raven-only `raven_mcp`, keeping full MCP schemas out of the main OpenCode session.
- **Remote and stdio MCP support** ‚Äî on-demand MCPs support remote HTTP/SSE servers and local stdio servers.
- **On-demand MCP metadata** ‚Äî Raven fills missing `autoGeneratedDescription` and `autoGeneratedTools`, with `/raven mcp refresh [server]` and `/raven mcp detail full|minimized` commands.
- **Avoided MCP schema stats** ‚Äî `/raven stats` now shows `Avoided MCP schema load` separately from delegated session and all-time savings.
- **MCP failure notifications** ‚Äî Raven shows a warning toast after startup or manual refresh when on-demand MCPs fail to load.
- **Duplicate MCP warnings** ‚Äî Raven warns when the same MCP server name is configured globally in OpenCode and in `onDemandMcpServers`.
- **Help command** ‚Äî `/raven help` now lists all Raven commands.

### Changed
- **Bundled MCP defaults** ‚Äî Context7, Exa, and Grep.app now default to on-demand MCP entries instead of global OpenCode MCP injection.
- **Global MCP routing** ‚Äî global MCPs configured in OpenCode are now auto-routed by detected server name, so `routeMcpServers` is no longer user-facing config.
- **Raven prompt/guidance** ‚Äî guidance now distinguishes globally configured MCPs from on-demand MCPs and teaches Raven to use `raven_mcp` internally.
- **Global Raven files** ‚Äî Raven now stores global files under `~/.config/opencode/opencode-raven/`, with generated on-demand MCP metadata split out of `raven-config.json` into `autogenerated-on-demand-mcp-metadata.json`.
- **Loaded-only schema stats** ‚Äî failed or pending on-demand MCPs no longer count toward `Avoided MCP schema load`.
- **On-demand MCP health** ‚Äî `/raven` and `/raven mcp` now show loaded, failed, and pending MCPs based on current-run connection checks instead of stale generated metadata.
- **Shorter status output** ‚Äî `/raven` now points to `/raven help` instead of printing the full command list inline.
- **Clearer routing labels** ‚Äî `/raven` now distinguishes routed global MCPs from on-demand MCPs, which are always behind Raven.
- **Default routed tools** ‚Äî `websearch` is routed by default because it is a built-in OpenCode search tool.

## 2.0.1

### Added
- **Keyword tool routing** ‚Äî `routeToolKeywords` and `/raven route keyword ...` route tools whose names contain a configured keyword, catching suffix-style MCP names like `web_search_exa`.

### Changed
- **Stats formula** ‚Äî context saved now uses a balanced estimate: final Raven session tokens minus the first assistant turn's input/cache baseline, then minus the compact answer returned. This counts tool/web/MCP result context while avoiding Raven prompt/tool-schema overhead.

## 2.0.0

### Added
- **Configurable tool/MCP routing** ‚Äî Raven can now route arbitrary tools or MCP server prefixes through `raven_seek` using `routeTools`, `routeMcpServers`, or `/raven route ...` commands.
- **Bundled MCP toggle** ‚Äî `raven-config.json` now includes `allowBundledMCPServers` to enable or disable Raven's bundled Context7, Exa, and Grep.app MCP defaults while preserving any existing `opencode.jsonc` MCP settings.
- **Custom Raven instructions** ‚Äî `ravenInstructions` lets users append extra Raven-only prompt guidance for custom MCPs without editing package files.

### Changed
- **Hard rerouting, not soft guidance** ‚Äî Raven is now documented and configured as a hard tool/MCP blocker-rerouter. Configured tools are blocked for non-Raven agents and must go through `raven_seek`.
- **Generalized beyond search** ‚Äî Raven still defaults to search/fetch/docs/GitHub routing, but can route any MCP whose opencode tool names share a prefix.
- **Config migration** ‚Äî default routed tools/MCP server prefixes are added to `raven-config.json` only when those fields are missing, so user removals stay removed.
- **Bash routing visibility** ‚Äî search-like bash command routing is now controlled by `bash` in `routeTools`. Remove `bash` to allow commands like `rg`, `Get-ChildItem -Recurse`, and `dir /s` directly.
- **Default routing cleanup** ‚Äî removed `websearch` from default routed tools/MCP prefixes. Users who need it can add it manually with `/raven route tool add websearch` or `/raven route mcp add websearch`.
- **Compact reroute errors** ‚Äî blocked MCP/tool calls now strip null and empty args before showing the `raven_seek` retry hint, reducing failed-call context noise.

## 1.2.8

### Fixed
- **Accurate context saved stats** ‚Äî `/raven stats` now reflects actual model token usage from `session.messages()` (last assistant message tokens √ó 4), not string output length. Both `raven_seek` and direct `@Raven` use the same formula: `totalSessionTokens √ó 4 ‚àí prompt ‚àí output`.

### Changed
- **`/raven` output** ‚Äî now includes `Raven context saved` section with session and all-time totals.
- **`/raven stats` output** ‚Äî renamed from "context processed" to "context saved" with clearer labeling.

## 1.2.7

### Changed
- **More reliable update notifications** ‚Äî startup update toasts are now shown from the event hook after the TUI event stream is active instead of during config loading.
- **`/raven` default output** ‚Äî now shows the loaded Raven package version and npm update availability.

## 1.2.6

### Fixed
- **Piped bash output filters** ‚Äî commands like `command | grep ...`, `command | rg ...`, `command | findstr ...`, and `command | head ...` are now allowed as bounded output filters instead of being rerouted to Raven.
- **Raven recursion prevention** ‚Äî `raven_seek` is denied inside the Raven agent so Raven cannot recursively call its own wrapper tool.
- **Model recovery after blocked tools** ‚Äî blocked tool errors now provide a short direct `raven_seek(query="...")` next-call hint using the attempted query/args.
- **Bash false positives** ‚Äî narrowed recursive/listing detection (`ls -r`, plain `cmd /c dir`, simple `Get-ChildItem`, comments, and bash descriptions no longer trigger blocking).

### Changed
- **Bash search detection** ‚Äî Raven still blocks primary discovery commands like `rg "foo" .`, `grep -R`, `find . -name`, `dir /s`, and `Get-ChildItem -Recurse`, but no longer treats simple pipeline filtering as a Raven-worthy search.
- **Raven guidance** ‚Äî `raven_seek` is described as the primary search/fetch/inspection tool, with explicit URL and command-output inspection support.
- **MCP defaults** ‚Äî Context7/Exa/Grep.app defaults now merge with existing user MCP config instead of overwriting custom headers, URLs, or disabled state.

## 1.2.5

- **Startup update notification** ‚Äî Raven checks npm at startup and shows a TUI toast when an update is available.
- **Config auto-migration** ‚Äî existing `raven-config.json` files are normalized on startup so newly-added default fields are populated automatically.

### Changed
- **Update docs** ‚Äî README now includes `/raven update`, manual `bun`/`npm` update commands, and opencode cache-clearing instructions.

- **External directory access** ‚Äî `external_directory: allow` prevents silent permission-prompt hangs in `raven_seek` sessions.

## 1.2.0

### Added
- **Agent allowlist** ‚Äî `excludeAgents` in config lets specific agents bypass search tool blocking and prompt injection (case-insensitive).
- **WebFetch/fetch blocking** ‚Äî `webfetch` and `fetch` tools are now intercepted, routing all web page fetches through Raven.
- **Reasoning effort config** ‚Äî `reasoning_effort` in config overrides Raven.md default. `/raven effort <value>` command.
- **Error recovery in raven_seek** ‚Äî rate limits, quota exhaustion, token overflow, model unavailable, and timeouts return actionable recovery guidance instead of raw errors.
- **Parallel search in Raven** ‚Äî Raven.md instructs Raven to run independent tool calls in parallel within a single session.
- **Global config** ‚Äî config moved to `~/.config/opencode/raven-config.json`, shared across all projects. Auto-creates on first run.

### Changed
- **raven_seek is primary path** ‚Äî all guidance now points to `raven_seek` only. `task(subagent_type="raven")` delegation path removed from instructions.
- **Subagent prompt injection** ‚Äî `<raven_guidance>` injected into every non-Raven, non-excluded subagent prompt at spawn time via `task`/`subtask` hooks.
- **Throttled error messages** ‚Äî full error shown once per session, silent on repeats. Prevents context flooding.
- **Improved error message** ‚Äî clean `REROUTE_MSG` with actionable `raven_seek` path.

### Fixed
- **Bash false positives** ‚Äî quoted content stripped before regex matching. `echo "use grep here"` no longer triggers blocking.
- **Allowlist reliability** ‚Äî `sessionAgents` map tracks session‚Üíagent for reliable allowlist lookups (vs `input.agent` which isn't populated on `tool.execute.before`).

### Removed
- **Dead `tool.execute.after` nullify code** ‚Äî replaced with stub for future analytics.
